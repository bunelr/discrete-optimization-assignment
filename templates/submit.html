Formating for C/C++

Input/Outputs

Your code should look for the metroEdgeDist.txt in the current folder.
For example using:
string filename = "metroEdgeDist.txt";
ifstream instream(filename.c_str());
The same way, you should output all your files in the current directory: all the dijkstra%d.txt, bf%d.txt, fw.txt and runtime.txt

Optional but strongly recommanded:
Your code should not always create the files. Mainly because it is very slow.
Your code should use the definition of the DOPRINT variable to know if you should or not create all the files. Use it the same way you use include guards:

bool print = false;
#ifdef DOPRINT
print = true;
#endif

Then in the rest of your code you can use this "print" variable to know if you should print the files or not. This can be done in any file, this DOPRINT will be defined for all your .hpp and .cpp files by our compilation script.

Warning: the runtime.txt should be outputed all the time even when the DOPRINT variable is not defined.

Code in the zip file

All your code should be in the main folder of the zip file.
For example, the zip should contain:
./main.cpp
./dijkstra.hpp
./dijkstra.cpp

Formating for python

Your code should contain at least one file called main.py .
This file should contain a function called "run".
If you want not to create all the files when timing your execution, that call function shoud take one argument which will be 0 if you shoud not print and 1 if you shoud print.

Warning: the runtime.txt should be outputed all the time even when the argument is set to 0.

The zip folder should contain all your files in the main folder.
It should look like:
./main.py
./dijkstra.py
